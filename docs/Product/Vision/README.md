# Vision Produit

Version : 1.0

Projet : ADCI

---

# Présentation

ADCI (Aujourd'hui, Demain, la Côte d'Ivoire) est une plateforme web permettant la gestion des adhésions au mouvement.

Le système permet aux visiteurs de consulter les informations relatives au mouvement, d'effectuer une demande d'adhésion en ligne et de suivre l'évolution de leur dossier.

---

# Contexte

Le mouvement ADCI souhaite disposer d'une plateforme moderne afin de digitaliser son processus d'adhésion.

La plateforme remplace un processus manuel par un parcours entièrement numérique.

---

# Problématique

Le processus actuel présente plusieurs limites :

- formulaires papier ;
- traitement manuel ;
- erreurs de saisie ;
- suivi difficile des adhérents ;
- absence de centralisation.

---

# Vision

Proposer une plateforme numérique simple, moderne et sécurisée permettant de gérer l'ensemble du processus d'adhésion au mouvement ADCI.

---

# Objectifs

Les objectifs principaux sont :

- simplifier l'adhésion ;
- améliorer l'expérience utilisateur ;
- réduire les traitements manuels ;
- sécuriser les données ;
- faciliter la gestion administrative.

---

# Valeur métier

La plateforme doit permettre :

- une meilleure qualité des données ;
- un gain de temps ;
- une diminution des erreurs ;
- une meilleure traçabilité ;
- une gestion centralisée des adhérents.

---

# Public cible

Le produit s'adresse principalement :

- aux visiteurs ;
- aux futurs adhérents ;
- aux administrateurs ;
- aux gestionnaires des adhésions.

---

# Fonctionnalités MVP

Le MVP comprend :

- consultation de la page Adhésion ;
- saisie des informations personnelles ;
- ajout de la pièce d'identité ;
- ajout facultatif du conjoint ;
- choix d'une formule ;
- récapitulatif ;
- soumission de la demande ;
- interface responsive.

---

# Hors périmètre V1

Ne sont pas inclus :

- paiement en ligne ;
- espace adhérent ;
- renouvellement d'adhésion ;
- tableau de bord administrateur ;
- notifications SMS ;
- signature électronique ;
- génération de carte d'adhérent ;
- authentification.

Ces fonctionnalités seront étudiées dans les versions ultérieures.

---

# Parties prenantes

- Bureau ADCI

- Développeurs
- Tech Lead

- Testeurs
- Futurs adhérents

---

# Critères de succès

Le MVP est considéré comme réussi lorsque :

- un visiteur peut réaliser une demande complète ;
- les données sont enregistrées ;
- une confirmation est envoyée ;
- le parcours est fluide sur ordinateur et mobile.
